## Appearance

### Default

Use `default` for the default dropdown menu appearance. The default menu will scroll after its height exceeds the pre-defined amount.

<Example
  Component={DropdownMenuDefault}
  packageName="@atlaskit/dropdown-menu"
/>

### Tall

Use `tall` to control the height of the menu. The tall menu will not scroll until the height exceeds the height of the viewport.

<Example Component={DropdownMenuTall} packageName="@atlaskit/dropdown-menu" />

## Should fit container

Use `shouldFitContainer` to fit the width of the dropdown menu to its parent width.

<Example
  Component={DropdownMenuShouldFitContainer}
  packageName="@atlaskit/dropdown-menu"
/>

## Custom triggers

There are two recommended ways to customize a trigger.

### Using triggerType and buttonProps

Set `triggerType` to `button` and customize the button using `buttonProps`.

<Example
  Component={DropdownMenuCustomTriggerButton}
  packageName="@atlaskit/dropdown-menu"
/>

### Using trigger

Use the `trigger` prop and pass in a custom trigger.

## Positioning

### Default positioning

By default the dropdown menu will be positioned next to your trigger.

<Example
  Component={DropdownMenuPositionDefault}
  packageName="@atlaskit/dropdown-menu"
/>

### Position

Use `position` to set the menu position, for example bottom right.

<Example
  Component={DropdownMenuPosition}
  packageName="@atlaskit/dropdown-menu"
/>

### Boundaries element

Use `boundariesElement` to keep the menu inside the viewport.

<Example
  Component={DropdownMenuBoundariesElement}
  packageName="@atlaskit/dropdown-menu"
/>

### Is menu fixed

Use `isMenuFixed` to determine if the menu should be in a fixed position. This is useful for breaking out of overflow scroll or hidden containers.
Extra layout management is needed to control scroll behavior when this property is enabled as the menu will not update the position with the target on scroll.

<Example Component={DropdownMenuFixed} packageName="@atlaskit/dropdown-menu" />
